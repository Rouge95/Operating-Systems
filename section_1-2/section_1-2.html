Operating System Structure, Services, and System Calls

Key Concepts

Java Virtual Machine (JVM)

Executes Java bytecode, providing platform independence and runtime management. Ensures programs run the same across devices.

Just-in-Time (JIT) Compiler

Converts bytecode to native machine code at runtime to improve execution speed.

Kernel

The core of the OS responsible for managing CPU, memory, I/O, and system calls. Acts as the bridge between hardware and software.

Layered Approach

OS is organized in layers; each layer provides services to the one above and uses services from the layer below. Easier maintenance and debugging.

Mach Operating System

An OS example using the microkernel approach for modularity and portability.

Microkernel Approach

Minimal kernel with essential services only. Other services run in user space for better reliability and security.

Modular Kernel

Kernel supports dynamically loadable modules to extend functionality without rebooting the system.

Operating-System Structure

Defines how the OS components, services, and interfaces are organized.

Simulation, Virtual Machine, and Virtualization

Techniques to run multiple OS instances or environments on a single physical machine.

VMware Architecture

An example virtualization platform using hypervisors to host multiple guest OSs.

Operating-System Services

Application Programming Interface (API)

Provides routines and protocols for programs to request OS services.

Services Helpful to Users

  • Command interpreter (shell) – Executes user commands.
  • Communications – Supports networking and messaging.
  • Error detection – Reports hardware or software issues.
  • File-system manipulation – Access, modify, and organize files.
  • Graphical User Interface (GUI) – Provides visual interface for users.
  • I/O operations – Facilitates input/output with devices.
  • Program execution – Loads, runs, and manages user programs.

Services for Efficient System Operation

  • Accounting – Tracks resource usage.
  • Protection and security – Ensures only authorized access.
  • Resource allocation – Manages CPU, memory, and device usage.

System Call

A controlled mechanism for user programs to request OS services. Switches CPU from user mode to kernel mode to execute privileged operations.

System-Call Interface

The interface that maps high-level API calls to kernel operations.

Systems Programs

  • Communication programs (e.g., web browsers) – Facilitate network operations.
  • Compilers, interpreters, assemblers, and debuggers – Translate and debug code.
  • File managers and text editors – Provide file operations and editing tools.
  • Logging, registry, system information tools – Help maintain and monitor system state.

Types of System Calls

  • Process control – create, terminate, or manage processes.
  • File management – open, read, write, close files.
  • Device management – request I/O operations, allocate devices.
  • Information maintenance – get/set system data, time, or status.
  • Communications – send/receive messages between processes or systems.

Reflection Questions with Answers

Understanding OS Services & System Calls

  • Three advantage points: usability (user-friendly services), efficiency (resource management), portability (hardware abstraction).
  • Performance services: resource allocation, accounting, protection/security. Usability services: GUI, command interpreter, file-system operations, program execution.
  • System calls: Mechanism for user programs to request kernel services. Works by switching CPU from user mode to kernel mode, executing privileged operations, then returning to user mode.
  • Linux system calls vs C library: The C library wraps system calls in standard functions, providing a higher-level, portable interface while the actual syscall executes in the kernel.

Kernel Structure Reflection

  • Microkernels move services to user space for reliability; layered kernels organize functionality hierarchically; modular kernels allow dynamic extension. Microkernels are smaller but may have performance overhead due to more inter-process communication.
  • Separating kernel and user space protects system stability, prevents accidental or malicious interference, and enforces privilege boundaries.
  • In virtual machines, user-mode programs and guest OS kernel calls are managed by the hypervisor. Privileged instructions are trapped and handled safely to emulate hardware behavior.
